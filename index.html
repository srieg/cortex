How Cortex works

A five-phase pipeline that produces a single, self-contained HTML file. No server. No database. Just open the file.

Phase 1
Research
Multi-agent deep research with source verification and content hashing
Phase 2
Outline
Structured outline with claim extraction and source mapping
Phase 3
Write
Prose generation with inline claim annotations and reasoning capture
Phase 4
Verify
Independent agent audits all claims, sources, and reasoning chains
Phase 5
Assemble
Package everything into a single HTML file with SHA-256 integrity hash

Inside the evidence store

Every Cortex paper embeds a structured JSON evidence store. Here's what's in it.

Claims

Every verifiable statement in the paper, typed by category with a confidence score and linked reasoning chain.

  • empirical
  • definitional
  • analytical
  • synthesis
  • methodological
  • normative

Reasoning Chains

The step-by-step thought process behind each claim — what the AI searched, evaluated, interpreted, and concluded.

  • query
  • evaluation
  • interpretation
  • inference
  • synthesis
  • decision
  • caveat

Sources

Full citation metadata with verification status, content hashes, relevant excerpts, and reliability ratings.

  • journal_article
  • preprint
  • dataset
  • book
  • government_data
  • institutional_report

Validations

Independent verification runs that audit every claim, check every source URL/DOI, and flag logical issues.

  • source_exists
  • claim_supported
  • logic_valid
  • confidence_justified
  • excerpt_accurate
  • normative_flagged

Under the hood

A Cortex paper is just an HTML file with an embedded JSON evidence store. Here's the core schema.

interface CortexDocument { meta: CortexMeta; // Title, authors, hash, abstract sections: Section[]; // Document structure with claim-annotated content claims: Record<string, Claim>; // Every verifiable statement reasoning: Record<string, ReasoningStep>; // How each conclusion was reached sources: Record<string, Source>; // Citations with verification status validations: ValidationRun[]; // Independent audit results } // Claims carry their own evidence graph: interface Claim { id: string; text: string; // The claim as stated in the paper claim_type: ClaimType; // empirical | definitional | analytical | ... confidence: number; // 0.0 - 1.0 reasoning_chain: string[]; // Ordered IDs into reasoning store source_refs: string[]; // IDs into source store alternatives_considered: Alternative[]; // What else was considered and why rejected } // The full schema is open and extensible. // See: Projects/cortex/schema/cortex-schema.ts

Use cases

Anywhere claims need evidence and reasoning needs to be visible.

Scientific Papers

Publish research where every empirical claim links to its source, and every inference shows its reasoning chain. Readers verify directly.

AI-Generated Reports

When AI writes, Cortex captures how it reached each conclusion. No black box. Full reasoning transparency.

Due Diligence

Investment memos, technical assessments, and compliance reports where every number needs a verifiable source trail.

Policy Analysis

Policy briefs where normative claims are explicitly flagged, and empirical claims are backed by government data and research.

Grant Proposals

Scientific grant applications where reviewers can click any claim to see the evidence base and assess feasibility.

Educational Materials

Textbooks and course materials where students can explore the reasoning behind every statement, building critical thinking skills.

Open format. Open source.

Cortex is a file format, not a platform. Generate papers with the CLI, build your own pipeline, or hand-craft the JSON evidence store. The schema is open and extensible.
